# Grad_judje
共創情報コース向けの卒業要件自己判定用スクリプト(非公認)

共創情報学コース 履修判定ツール README
=====================================

1. 概要
-------
このツールは、室蘭工業大学大学院・情報電子工学系専攻・共創情報学コース向けに、
履修登録状況CSVを読み取り、修了要件の達成状況を判定するための Python スクリプトです。

主な機能:
- 履修登録CSVから科目名を抽出
- 必修・選択・自コース・専攻共通・他コース・副専修などの区分で単位を集計
- 修了要件ごとの OK / NG を表示
- 未取得の必修科目一覧を表示
- 指定した学年・クォータで、今期取得を推奨する科目を表示
- 未分類科目を表示して、科目マスタの追加漏れを見つけやすくする

注意:
- このツールは、大学の公式判定を代替するものではありません。
- 最終的な修了判定や科目区分の扱いは、必ず履修要項・最新のガイダンス資料・教務担当の案内を確認してください。


2. 想定ファイル
---------------
- スクリプト本体: judge_tool.py
  （配布版では co_creation_requirements_checker_v3.py などの名前でも可）
- 入力CSV: CAMPUS SQUARE 等から出力した履修登録状況CSV
   (履修登録ページ下部の"テキスト出力"から取得)

例:
- judge_tool.py
- RSReferCsv.csv


3. 必要環境
-----------
- Python 3.10 以上推奨
  （dataclass / argparse / pathlib を利用）
- 追加ライブラリ不要
  （標準ライブラリのみで動作）


4. 事前準備
-----------
(1) Python をインストールする
(2) judge_tool.py を任意のフォルダに置く
(3) 履修登録状況CSVを同じフォルダに置く
    もしくは、実行時にCSVのフルパスを指定する


5. 実行方法
-----------
基本形:

    python judge_tool.py "RSReferCsv.csv" --year 1

学年とクォータを明示する場合:

    python judge_tool.py "RSReferCsv.csv" --year 1 --term 2Q

現在日付からクォータを自動推定する場合:

    python judge_tool.py "RSReferCsv.csv" --year 1 --term auto

2年次として判定する場合:

    python judge_tool.py "RSReferCsv.csv" --year 2 --term 3Q

venv を使う例:

    /path/to/.venv/bin/python judge_tool.py "RSReferCsv.csv" --year 1


6. 引数の説明
-------------
(1) csv_path
    読み込む履修登録CSVのパスです。

(2) --year
    現在の学年を指定します。
    指定可能な値:
    - 1
    - 2

    例:
    --year 1

(3) --term
    今期として扱うクォータを指定します。
    指定可能な値:
    - auto
    - 1Q
    - 2Q
    - 3Q
    - 4Q
    - INTENSIVE

    例:
    --term 1Q

    auto を指定すると、実行日からクォータを推定します。


7. 出力内容
-----------
主に次の内容が表示されます。

- 抽出された履修科目
- 単位集計
  - 必修
  - 選択
  - 合計
  - 自コース＋専攻共通
  - 他コース・他専攻
  - 副専修（他コース）
  - 副専修（全学共通）
- 修了要件判定
- 未取得の必修科目
- 今期の取得推奨科目
- 未分類科目


8. よくあるエラー
-----------------
(1) CSVファイルが見つかりません
    指定したCSVファイル名またはパスが間違っています。

    対処:
    - ls でファイル名を確認する
    - フルパスを指定する
    - ファイル名の末尾に (1) などが付いていないか確認する

(2) --year が必要です と表示される
    --year 1 または --year 2 を付けて実行してください。

(3) 引数を付けたのに認識されない
    半角スペースではなく全角スペースが混ざっている可能性があります。

    誤:
    "RSReferCsv.csv"　--year 1

    正:
    "RSReferCsv.csv" --year 1

(4) 未分類科目が出る
    スクリプト内の COURSE_CATALOG に対象科目が登録されていません。
    科目マスタを追記してください。


9. 重要: 他コース履修科目は自分で追加が必要な場合が多い
---------------------------------------------
このツールを他の学生が使う場合、最も重要なのがこの点です。

共創情報学コースでは、他コース履修や副専修科目の組み方が人によって異なりやすく、
特に次のような科目は、その人の履修計画に応じて COURSE_CATALOG に追加・修正が必要になる場合が多いです。

- システム情報学コースの科目
- 他専攻のコース科目
- 副専修の全学共通科目
- 個別に認められた他コース履修科目
- 年度によって開講時期が変わる科目

そのため、他コース履修を使う人は、必ず以下を確認してください。

(1) その科目が本当に「他コース・他専攻」の要件に算入できるか
(2) その科目が副専修の「他コース履修科目」として算入できるか
(3) 自コース開講科目と同一扱いで除外されないか
(4) 開講時期（1年1Q / 2年1Q / 集中講義 など）が正しいか

特に、他コース履修は利用者ごとに対象科目が大きく変わるため、
配布時の初期状態だけで完全には対応できないことが多いです。
「未分類科目」が出たら、その都度 COURSE_CATALOG を更新してください。


10. 科目マスタの編集方法
------------------------
スクリプト内には、科目名ごとの設定をまとめた COURSE_CATALOG があります。
新しい科目を追加したい場合は、同じ形式で追記してください。

例:

    "信号処理特論": CourseInfo(
        2,
        "cross_specialty",
        offered_slots=slots((1, "1Q"))
    ),

主な bucket の意味:
- mandatory         : 必修
- self_course       : 自コース科目
- common            : 専攻共通科目
- pbl               : PBL・社会連携科目
- cross_specialty   : 情報×専門科目など、他コース系で集計したい科目
- other_course      : 副専修の他コース履修として直接扱う科目
- all_university    : 副専修の全学共通科目


11. 開講時期の設定方法
----------------------
開講時期は、学年つきで設定します。

例:
- Y1_1Q         = 1年1Q
- Y1_2Q         = 1年2Q
- Y1_3Q         = 1年3Q
- Y1_4Q         = 1年4Q
- Y1_INTENSIVE  = 1年集中講義
- Y2_1Q         = 2年1Q
- Y2_2Q         = 2年2Q
- Y2_3Q         = 2年3Q
- Y2_4Q         = 2年4Q

設定例:

    offered_slots=slots((1, "2Q"))

通年科目の例:

    offered_slots=full_year_slots(1)

2年次の通年科目の例:

    offered_slots=full_year_slots(2)

注意:
開講時期を誤ると、今期推奨に 2年次のみの科目が 1年次に表示されるなどの問題が起こります。
必ず年度の履修要項・ガイダンスと照合してください。


12. 判定仕様に関する注意
------------------------
このツールでは、科目区分の判定を COURSE_CATALOG に依存しています。
したがって、次のような場合は結果が実情とずれる可能性があります。

- 年度によって科目区分が変更された
- 開講クォータが変更された
- 他コース履修として使えるかどうかの扱いが年度で異なる
- 同一名称でも算入区分が異なる
- 教務上の例外運用がある

そのため、配布先の利用者には必ず以下を案内してください。

- 未分類科目が出たら、まず科目マスタを見直す
- 他コース履修科目は自分の履修要項・ガイダンスで確認する
- 最終的な修了可否は教務や指導教員にも確認する


13. おすすめの運用
------------------
- まず一度実行して、未分類科目を確認する
- 自分が履修予定の他コース科目・副専修科目を追加する
- 再実行して判定と推奨結果を確認する
- 学年やクォータを変えて、先の学期の推奨も確認する

例:

    python judge_tool.py "RSReferCsv.csv" --year 1 --term 1Q
    python judge_tool.py "RSReferCsv.csv" --year 1 --term 2Q
    python judge_tool.py "RSReferCsv.csv" --year 1 --term 3Q


14. 免責
--------
このツールの判定結果は参考情報です。
最終的な修了要件の判定、科目区分、履修可否、開講時期の確認は、
必ず最新の履修要項・ガイダンス・教務担当の案内に従ってください。
